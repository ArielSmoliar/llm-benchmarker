Product Requirements Document

LLM Model
Benchmarker

A no-code tool for comparing AI model outputs side-by-side — built for product teams who want to spend more time building AI products, less time wiring up APIs.

Draft v0.2 April 2026 Open Source · GitHub Self-hosted · Docker
01 — Context

Problem & Opportunity

Problem

Choosing between LLMs today requires running manual experiments across different SDKs, API formats, and dashboards. There's no fast, frictionless way to compare model outputs, latency, and quality in one place.

Opportunity

NVIDIA now hosts ~80 open-source models behind a single OpenAI-compatible API — for free. This collapses the switching cost to a single endpoint change, making a model benchmarker both easy to build and genuinely useful.

02 — Audience

AI Product Builders

📋
Product managers
Deciding which model to ship in a feature without depending on engineering. Need clear output quality and latency signals, not code.
🎨
Designers & content strategists
Evaluating tone, style, and output quality across models for copywriting, UX writing, or generative content features. Care about feel, not benchmarks.
📊
Data scientists & analysts
Running structured comparisons across prompts and models to inform team decisions. Want exportable results without building eval pipelines from scratch.
03 — Scope

Goals & Non-Goals

The tool must make it trivially easy to run the same prompt against multiple models and compare results across output quality, latency, and token usage — all in one screen.

It should be runnable locally in under 2 minutes via docker-compose up and shareable on GitHub without any cloud setup.

No accounts. No signup. No data leaving the user's machine. The only credential required is a free NVIDIA API key — and that stays in a local .env file.

Not a fine-tuning or training tool

Not a production LLM router or proxy

Not a SaaS product — no accounts, no billing, no hosted version

Not limited to NVIDIA (any OpenAI-compatible endpoint should work)

No user accounts required — runs entirely on the user's machine

No data leaves the user's machine — prompts and results stay local

07 — Roadmap

Delivery Phases

Phase 1 — MVP
Get it on GitHub
  • Dynamic model list from NVIDIA API (chat models only)
  • Checkbox selector (up to 5) grouped by provider
  • Prompt input + parallel API calls via FastAPI
  • Result cards with latency + token count
  • Copy button per card; latency bar chart
  • docker-compose.yml + README with setup instructions
Phase 2 — Worth Starring
Eval layer + community hooks
  • Human eval — thumbs up voting per card; winner highlighted
  • LLM-as-judge — "Auto-evaluate" button scores responses 1–5 on accuracy, clarity, conciseness
  • Judge model is user-selectable; prompt is transparent and editable
  • Run history saved in localStorage (includes eval scores)
  • Shareable URL encoding prompt + results + scores
  • System prompt support
Phase 3 — Power Users
Depth & data
  • Temperature / max_tokens controls per model
  • Dataset / batch eval — upload CSV of prompts, aggregate judge scores per model
  • SQLite persistence + CSV export of eval results
  • Support for additional OpenAI-compatible endpoints
  • Streaming responses with time-to-first-token tracking
